    Descrição do caso escolhido
- O projeto consiste em uma plataforma institucional voltada para a apresentação e oferta de serviços de "Servidor na Nuvem". A iniciativa foca em atender pequenas e médias empresas, além de profissionais de TI autônomos, que necessitam de infraestrutura escalável, segura e de alta disponibilidade para hospedar seus sistemas, bancos de dados e aplicações sem a necessidade de manter hardware físico local.

   Necessidade identificada
- Identificou-se a ausência de uma presença digital clara e otimizada para o serviço. Havia a necessidade de estruturar uma página que não apenas explicasse o funcionamento técnico do servidor na nuvem, mas que também corrigisse falhas graves de SEO (Search Engine Optimization), melhorando o ranqueamento nos mecanismos de busca do Google. Além disso, faltava uma identidade visual profissional e um layout totalmente responsivo e reestruturado para transmitir credibilidade ao mercado.

   Público-alvo
- O público-alvo é composto por gestores de tecnologia (CTOs), proprietários de pequenas e médias empresas e desenvolvedores autônomos. Esse público precisa encontrar com extrema facilidade informações sobre a segurança dos dados, planos de escalabilidade, desempenho do servidor, canais de suporte técnico e, principalmente, uma tabela ou área clara com os preços e especificações dos servidores.

   Objetivo do site
- O objetivo principal é a transmissão de uma mensagem clara, objetiva e profissional para o usuário (o potencial contratante), gerando confiança imediata para que ele realize a ação principal da página: entrar em contato com a equipe comercial para solicitar um orçamento personalizado ou escolher um plano inicial de contratação.

   Processo de desenvolvimento
- O desenvolvimento foi estruturado a partir de um diagnóstico inicial focado nas dores do cliente de infraestrutura de TI. Iniciamos desenhando uma estrutura semântica que priorizasse a hierarquia de tópicos. Durante o processo, refinamos o layout para garantir que a transição entre telas grandes e dispositivos móveis ocorresse sem quebras de design, reestruturando seções de blocos de texto densos para elementos visuais mais leves (cards) e aplicando otimizações de SEO diretamente nas tags estruturais para garantir relevância no Google.

   Principais decisões do projeto
- Estrutura HTML e SEO: Aplicação rigorosa de tags semânticas (header, main, section, footer) e organização lógica de títulos (H1, H2, H3) para o aprimoramento de SEO da página, facilitando a indexação do Google. Imagens ganharam atributos 'alt' descritivos com palavras-chave relevantes.
- Identidade Visual e Layout: Escolha de uma paleta de cores moderna (focada em tons de azul e grafite), que transmite tecnologia, segurança e estabilidade. O layout foi totalmente reestruturado para ser limpo e agradável, evitando excessos visuais e garantindo excelente contraste para leitura.
- Responsividade: Uso de media queries e do sistema de grid responsivo para assegurar que a página se adapte perfeitamente a smartphones e desktops, eliminando barras de rolagem horizontais e mantendo os botões de ação acessíveis em qualquer tamanho de tela.
- Acessibilidade: Navegação estruturada linearmente, permitindo o uso básico por teclado, manutenção de alto contraste entre fundos e textos, e tags semânticas para leitores de tela.


  Funcionalidade em JavaScript: 
- O JavScript foi utilizado para a navbar seguir o scroll do usuário na página, e junto ao css, foi colocado um efeito de opacidade.

  Uso de Bootstrap:
- Foi utilizado o componente “navbar” para criar um cabeçalho que gera automaticamente um "menu hambúrguer" quando aberto no celular.
- Para organizar o conteúdo lado a lado no computador e empilhado no celular, utilizamos o container e o sistema de linhas e colunas (grid responsivo). Isso foi feito em praticamente todas as seções (<section>).
-Foram usadas classes utilitárias para espaçamentos (padding/margin), alinhamento de texto e formatação visual dos botões.

  Testes realizados
- foram feitos testes de responsividade em diferentes telas, utilizando a ferramenta de inspecionar do navegador, e checando se o código retornaria algum erro referente ao código javascript no console. Além disso, o código HTML também foi checado no site validador.w3.org.
